# 开始这里

这个仓库现在的主线不是继续 polish 文章，而是通过 MiniClaw 原型理解 Harness Agent。

你要掌握的问题只有一个：

> 一个 Agent 如何从“会回答”变成“能持续行动、观察环境、管理上下文、恢复任务、输出证据”的系统？

## 先跑一条命令

```bash
make verify
```

或者直接跑脚本：

```bash
scripts/verify_offline.sh
```

这会验证当前学习系统是否完整：

- MiniClaw 测试
- minimal harness 测试
- MiniClaw walkthrough 证据生成
- whitespace 检查

如果这条命令通过，说明本地离线学习环境是健康的。

## 再生成一次证据包

```bash
make walkthrough
```

或者直接跑脚本：

```bash
scripts/run_miniclaw_walkthrough.sh
```

脚本会输出一个 `walkthrough-output/miniclaw-<timestamp>/` 目录。重点看这些文件：

- `仓库分析-trace.txt`：观察 Agent Loop 如何从 plan 到 tool_call 再到 observation。
- `仓库分析-state.txt`：观察上下文如何沉淀成可恢复状态。
- `memory.txt`：观察完成结果如何进入长期记忆。
- `codeact-trace.txt` 和 `codeact-state.txt`：观察 CodeAct 如何通过受限工具执行。
- `approval-trace.txt`：观察 Human Approval 如何成为系统状态。
- `compact-trace.txt`：观察长 trace 如何被压缩。
- `task-report.md`：观察一次任务如何变成可复盘证据。

## 快速做一次学习验收

```bash
make learn-check
```

这会创建临时 workspace，自动检查最小学习闭环：

- Agent Loop 是否出现 `plan -> tool_call -> observation -> final_result`
- CodeAct 是否记录 `code_safety_status`
- CodeAct 是否记录 `imports=blocked`、`builtins=empty` 执行边界
- Memory 是否保存 `repo_analysis` 长期记忆
- Compact 是否把长 trace 摘要进 `compact_summary`
- Boundary 是否暴露 `shell=False`、`cwd=workspace`、`allowlist=matched`
- Human Approval 是否给出 `approve-task` 下一步
- Task Report summary 是否能生成学习摘要

## 什么时候算真的学到

不要用“读完了几章”判断。用下面的问题判断：

- Trace 和 State 分别解决什么问题？
- Skill 为什么是渐进加载，而不是普通提示词模板？
- SubAgent 隔离了什么噪声？代价是什么？
- CodeAct 的风险为什么必须放在 Harness 工具边界里控制？
- Memory 和聊天历史有什么区别？
- Compact 后系统还保留了哪些恢复任务的证据？
- 为什么 Harness Agent 的关键不只是模型，而是模型外部的行动系统？

能回答这些问题，再去写文章就不会只是“形式上有深度”。

## 真实模型验证

离线测试默认不会调用真实模型。配置好 key 后再跑：


它只跑 real-model smoke：MiniClaw runtime、planner、CodeAct，以及 minimal harness Plan-Act。

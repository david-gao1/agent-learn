# Plan-Act

## 解决什么问题

复杂任务如果直接行动，容易遗漏步骤、来回跳转或过早陷入细节。Plan-Act 解决的是任务拆解和执行顺序控制问题。

## 核心机制

先生成一个短计划，明确目标、步骤和完成条件；执行时逐步推进，并允许根据观察修订计划。

## 最小例子

分析仓库前先列出三步：查看结构、读取关键文档、总结架构。执行每一步时记录观察，最后合并结论。

## 和相近概念的区别

Plan-Act 比 ReAct 更强调开始前的结构化拆解。它不是瀑布计划，好的实现应允许计划被观察结果修正。

## 在 Harness Agent 原型中的位置

`TaskStore` 可以保存计划步骤和状态。后续 Plan-Act Demo 应让 Agent 先写计划，再按步骤调用工具。

# SubAgent

## 解决什么问题

复杂任务会产生大量中间观察，全部塞进主上下文会污染决策。SubAgent 解决的是复杂子任务的上下文隔离问题。

## 核心机制

主 Agent 把一个边界清晰的子任务交给独立上下文窗口执行。SubAgent 自己观察、行动、总结，只把压缩后的结果交回主 Agent。

## 最小例子

主 Agent 需要了解测试体系，就派一个 SubAgent 只阅读测试目录并返回“测试框架、关键用例、风险”三点摘要。

## 和相近概念的区别

SubAgent 不是普通函数调用。函数返回确定结果，SubAgent 内部也有自己的上下文和循环。它和 Compact 都能降低噪声，但 SubAgent 是隔离，Compact 是压缩。

## 在 Harness Agent 原型中的位置

当前原型还没有 SubAgent。后续 Demo 应创建一个隔离的仓库阅读循环，只把摘要写回主任务。

# 09 Memory

## 解决什么问题

上下文窗口有限，任务也可能跨会话。Memory 解决的是重要信息如何在当前窗口之外保存、检索和复用的问题。

## 核心机制

把信息分层保存：短期记忆保留最近消息和观察；长期记忆保存稳定事实、偏好、经验或任务状态；需要时再检索回上下文。

## 最小例子

Agent 记住“这个仓库的原型入口是 `demo.py`”，下次继续分析时不必重新从零探索。

## 和相近概念的区别

Memory 强调跨时间保存。Context 是当前调用能看到的信息。Task System 保存的是工作进度，可能属于结构化记忆的一部分。

## 在 Harness Agent 原型中的位置

当前原型的 `TaskStore` 是任务状态记忆，`compact_messages` 是短期上下文压缩。后续可增加长期事实存储。

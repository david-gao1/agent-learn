# Skills

## 解决什么问题

专家经验如果每次都塞进提示词，会浪费上下文；如果完全不提供，Agent 又缺少稳定方法。Skills 解决的是可复用专家上下文的渐进加载问题。

## 核心机制

把某类任务的方法、约束、脚本和参考资料打包成目录。先暴露标签和简介，选中后再加载 `SKILL.md`，需要时再读取更深资源。

## 最小例子

“代码仓库阅读 Skill”说明先看目录、README、测试、入口文件，再输出架构和风险。

## 和相近概念的区别

Skill 不是普通 prompt 模板。它更像可按需加载的工作包。RAG 主要检索资料；MCP 主要提供工具协议；Skill 管理的是方法和上下文。

## 在 Harness Agent 原型中的位置

`LocalSkillLoader` 先列出 Skill 标签，再加载 `repo-reading/SKILL.md`。这展示了 L1/L2 的渐进加载思路。

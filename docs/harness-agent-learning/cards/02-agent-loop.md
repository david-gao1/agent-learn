# 02 Agent Loop

## 解决什么问题

单轮问答只能生成一次答案，无法持续观察环境、采取行动、根据结果修正。Agent Loop 解决的是多步任务中的连续行动问题。

## 核心机制

循环执行：理解当前状态，选择下一步行动，调用工具或生成输出，读取观察结果，更新上下文或任务状态，再决定继续、重试或结束。

## 最小例子

用户要求分析一个项目。Agent 先列目录，再读取 README，再查看测试文件，最后整合报告。每一步都依赖上一轮观察。

## 和相近概念的区别

工作流通常是预先固定步骤。Agent Loop 可以根据中间观察动态选择下一步，但也需要更强的停止条件、错误处理和状态管理。

## 在 Harness Agent 原型中的位置

`HarnessAgent.run_workspace_analysis` 展示了离线版循环：创建任务、加载 Skill、观察文件、读取内容、压缩上下文、完成任务。

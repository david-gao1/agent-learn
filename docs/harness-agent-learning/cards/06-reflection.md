# 06 Reflection

## 解决什么问题

Agent 第一次输出经常不完整、证据不足或方向偏了。Reflection 解决的是结果质量检查和有限重试问题。

## 核心机制

在一次行动或生成之后，引入 critique 步骤：检查目标是否满足、证据是否足够、是否需要补充观察，并决定接受、修正或重试。

## 最小例子

Agent 输出仓库分析后，检查是否包含入口、测试、核心模块。如果缺少测试信息，就再次读取测试目录再修订报告。

## 和相近概念的区别

Reflection 不是简单重复调用模型，而是带着明确评价标准重看结果。它和 Plan-Act 可组合：每个计划步骤完成后都可反思。

## 在 Harness Agent 原型中的位置

`run_reflection` 展示了最小 Reflection：生成结果后执行 critique，如果发现缺口，就在有限次数内重试。

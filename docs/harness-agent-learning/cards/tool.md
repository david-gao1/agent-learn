# Tool

## 解决什么问题

模型只能生成文本，不能直接观察文件系统、执行命令、查询数据库或修改环境。Tool 解决的是模型和外部世界之间的行动接口问题。

## 核心机制

Harness 把外部能力包装成受控函数，定义输入、输出、安全边界和错误返回。模型或循环选择工具，Harness 执行工具并把结果写回上下文。

## 最小例子

`read_file("README.md")` 返回文件内容；`run_bash("find . -maxdepth 2 -type f")` 返回目录观察结果。

## 和相近概念的区别

Tool 是行动接口。MCP 是一种把工具暴露给模型客户端的协议。RAG 通常偏向检索知识，不一定能改变环境。

## 在 Harness Agent 原型中的位置

`tools.py` 中的 `run_bash` 和 `read_file` 是基础工具。它们让 Agent 通过文件和命令观察工作区。

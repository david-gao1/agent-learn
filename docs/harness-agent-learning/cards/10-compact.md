# 10 Compact

## 解决什么问题

Agent 长时间工作后，上下文会变长、重复、噪声增多。Compact 解决的是在不完全丢失进度的前提下释放上下文空间。

## 核心机制

把旧消息、工具观察和中间结论压缩成结构化摘要，同时保留最近的关键消息和当前任务状态。

## 最小例子

读取了十个文件后，不继续保留所有原文，而是压缩成“已查看文件、关键发现、待确认问题、下一步”。

## 和相近概念的区别

Compact 是压缩当前工作上下文。Memory 是跨会话保存和检索。SubAgent 是隔离子任务，减少主上下文接触噪声。

## 在 Harness Agent 原型中的位置

`compact_messages` 将旧消息合成一条摘要，并保留最近消息。它是原型里最小的上下文压缩机制。

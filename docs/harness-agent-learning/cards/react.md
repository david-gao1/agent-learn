# ReAct

## 解决什么问题

模型如果只推理不观察，容易在假设上继续编故事；如果只调用工具不判断，又缺少方向。ReAct 解决的是推理和行动脱节的问题。

## 核心机制

把过程组织成 Reasoning、Action、Observation 的交替循环：先判断需要知道什么，再行动获取信息，再根据观察更新判断。

## 最小例子

分析失败测试时，Agent 先判断需要看测试输出，执行测试命令，读取错误，再决定查看相关代码。

## 和相近概念的区别

ReAct 强调每一步的思考、行动、观察闭环。Plan-Act 更强调先拆出计划；Reflection 更强调对已有结果进行批判和重试。

## 在 Harness Agent 原型中的位置

当前原型已经有简化的观察和行动顺序。后续可以把每一步记录成 `reason -> action -> observation`，作为 ReAct Demo。

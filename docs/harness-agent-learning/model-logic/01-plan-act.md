# Plan-Act

## 这是什么

Plan-Act 是一种把 Agent 任务拆成“先规划，再执行”的模式。它不要求计划一次写死，而是先给 Agent 一个可执行的路线，让后续行动不再只是凭当前感觉往前试。

最小形式是：

```text
目标 -> 计划步骤 -> 执行一步 -> 记录观察 -> 继续下一步或修正计划
```

## 想解决什么问题

它主要解决复杂任务里的失控感。

如果没有计划，Agent 很容易出现三个问题：

- 一上来就执行细节，忘了整体目标。
- 做了很多动作，但不知道是否接近完成。
- 中间观察变多以后，下一步选择越来越随意。

Plan-Act 的核心价值不是“计划一定正确”，而是让行动有一个可检查的结构。

## 原理是什么

Plan-Act 把一个任务拆成两个控制层：

- **Plan**：把目标变成少量步骤，并定义大致完成条件。
- **Act**：按步骤调用工具、读取观察、更新状态。

关键点是：计划不是替代观察，而是约束观察。每一步行动都应该回答一个问题：这一步是否减少了完成任务所需的不确定性？

一个好的 Plan-Act 循环通常包含：

```text
目标定义 -> 步骤拆解 -> 执行当前步骤 -> 观察结果 -> 更新步骤状态 -> 判断是否继续
```

## 如何实践

练习场景：让 Agent 分析一个小项目。

先不要写代码，先写出三步计划：

1. 查看项目文件结构。
2. 阅读 README 或入口文件。
3. 总结项目用途、核心模块和风险。

验收信号：

- 每一步都有明确观察结果。
- 最终总结能追溯到前面的观察。
- 如果某一步没有产生有效信息，能说明下一步如何调整。

# CodeAct

## 这是什么

CodeAct 是一种让 Agent 用代码表达行动的模式。它不是只让模型“说该怎么做”，而是让模型生成一段可执行代码，由 Harness 执行后把结果返回给 Agent。

最小形式是：

```text
任务 -> 生成代码 -> 受控执行 -> 获得结果 -> 根据结果继续决策
```

## 想解决什么问题

它主要解决自然语言行动不适合精确处理的问题。

有些任务如果只靠普通工具调用，会很笨：

- 批量统计文件。
- 解析结构化文本。
- 做小规模数据转换。
- 把多个基础操作组合成一次行动。

CodeAct 的价值是让 Agent 把“临时操作逻辑”写成代码，而不是等待系统提前提供所有工具。

## 原理是什么

CodeAct 的关键不是“能执行代码”，而是“代码执行必须由 Harness 管住”。

它至少包含四个部分：

- **代码生成**：模型根据任务写出一段小代码。
- **执行边界**：限制文件、网络、导入、时间和权限。
- **结果回传**：把 stdout、返回值或错误写回上下文。
- **后续判断**：Agent 根据执行结果决定是否继续。

最小控制规则是：

```text
能用固定工具完成 -> 优先固定工具
需要临时组合逻辑 -> 考虑 CodeAct
涉及高权限或不可逆动作 -> 禁止或人工确认
```

## 如何实践

练习场景：统计一个目录里 Markdown 文件数量。

先写出受限任务：

1. 只能读取当前 workspace。
2. 不允许 import。
3. 不允许打开任意路径写文件。
4. 执行后只能通过 `result` 返回结果。

验收信号：

- 代码能完成统计。
- 违规代码会被阻止。
- 执行边界能被复盘，例如 `imports=blocked`、`builtins=empty`。
- 输出结果能进入后续 Agent 判断。

## 对应原型

原型中的 `execute_codeact` 是最小实现：

- 文件：`prototypes/minimal_harness_agent/src/minimal_harness_agent/patterns.py`
- 测试：`test_codeact_executes_limited_python_and_blocks_imports`

它只用于学习 CodeAct 的基本结构，不是生产级安全沙箱。真正系统里还需要进程隔离、权限控制、超时、资源限制和审计。
